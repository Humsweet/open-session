# 命令速查 · open-session

> 给你的命令小抄。每条都告诉你：**这命令是什么 / 干什么用 / 啥时候用**。
> 怎么用：在这个文件夹里打开终端（Windows 用 PowerShell），把命令整行复制粘贴进去按回车。

## 这个项目是干啥的

一个跑在本地的网页应用：把你电脑上各种 AI 编程工具（Claude、Codex、Copilot、Gemini 等）的「会话记录（session，就是你跟 AI 的一次次对话）」收集起来，在一个网页界面里浏览、搜索、按项目过滤，还能帮你自动总结每次会话讲了啥。数据存在本地的小型数据库里。

它是用 Next.js + React 做的（Next.js = 一个流行的网页开发框架），所以靠 `npm` 这套命令来运行。它不是装到电脑里的软件，而是在本地起一个服务，然后你用浏览器打开来看。

## 🌟 最常用（先记这几条）

### ▶ 启动这个项目（你最常问的 "start this pj"）

```
npm run dev
```

- **干什么**：在本地起一个「开发模式」的网页服务，跑这个会话浏览器。
- **啥时候用**：想在本地把它打开来用、或者改了代码想看效果时（开发模式支持热重载——改完代码网页会自动刷新，不用重启）。
- **跑起来之后**：浏览器打开 **http://localhost:3000** 就能看到界面（在你这台 Windows 电脑上日常就是跑在 3000 端口）。这个终端窗口会一直挂着，**别关**，关了服务就停了。
- **怎么停**：在终端按 Ctrl + C。

> 端口小科普：「端口」就像门牌号，`localhost:3000` 表示「本机的 3000 号门」。这个项目开发模式用 3000，生产模式（见下面）用 3001，别搞混。

## 📋 全部常用命令

### 装依赖

```
npm install
```

- **是什么**：把项目运行需要的一堆第三方代码包下载安装到本地。
- **干什么**：准备好运行环境，装完会多出一个 `node_modules` 文件夹（很大是正常的）。
- **啥时候用**：第一次拿到项目、换电脑、或别人更新了依赖之后。装过一次日常就不用再装。

### 启动（开发模式，最常用）

```
npm run dev
```

- **是什么**：起开发服务器，跑在 http://localhost:3000。
- **干什么**：本地跑这个网页应用，改代码自动热重载。
- **啥时候用**：日常使用、或开发调试时。**这是你在 Windows 上最常用的一条。**

### 打包（生产模式编译）

```
npm run build
```

- **是什么**：把项目编译成「生产版本」（更精简、更快的正式版）。
- **干什么**：为正式上线/长期运行做准备。
- **啥时候用**：要用生产模式跑、或部署时，先跑这条编译一次。日常本地用开发模式（`npm run dev`）就够了，不需要 build。

### 启动（生产模式）

```
npm start
```

- **是什么**：跑已经 build 好的生产版本，端口是 **3001**。
- **干什么**：用更精简的正式版运行（必须先 `npm run build` 编译过才能跑）。
- **啥时候用**：正式长期运行时。本地日常调试用 `npm run dev` 就行，一般用不到这条。

### 检查代码规范

```
npm run lint
```

- **是什么**：用 ESLint 检查代码有没有写法问题（lint = 代码体检）。
- **干什么**：找出潜在的写法错误和不规范的地方。
- **啥时候用**：改完代码想确认没写出明显毛病时。日常使用不需要跑。

## 💡 小白提示

- **得先有 Node.js**：`npm` 是跟着 Node.js 一起装的。如果跑 `npm` 提示「找不到命令」，去官网（nodejs.org）下载安装 Node.js 即可。
- **第一次用，先 `npm install` 再 `npm run dev`**：装好依赖之后，日常只要 `npm run dev` 就能跑起来。
- **记住地址是 http://localhost:3000**：用 `npm run dev` 跑起来后，在浏览器输入这个地址就能看到界面。如果打不开，先看终端有没有报错、服务是不是还在跑。
- **开发模式 vs 生产模式别混**：日常本地用 `npm run dev`（端口 3000，改代码自动刷新）。`npm run build` + `npm start`（端口 3001）是给正式部署用的，本地一般不用。
- **启动那个终端别关**：服务跑起来后终端会一直挂着，关了服务就停。想停在那个终端按 Ctrl + C。
