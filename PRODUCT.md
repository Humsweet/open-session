## Users

A single developer (the repo owner) who runs multiple AI coding CLIs (Claude Code, Codex CLI, Copilot CLI, Gemini CLI) across two machines (Windows + macOS). They open this tool to review, search, and triage past agent sessions — typically mid-workflow, to recall what an agent did, pin important sessions, or generate summaries.

## Product Purpose

open-session is a local dashboard that scans the session logs of multiple AI CLI tools, normalizes them into one timeline, and makes them browsable: filter by tool/status/origin, read message-level detail (thinking, tool calls, results), pin and annotate. Success = finding any past session and understanding what happened in it within seconds.

## Brand Personality

Disciplined, tool-like, trustworthy. Visual language follows the Daily_Briefing design system (warm neutral surfaces, a single red accent `#C0392B`, desaturated earthy status colors, monospace for data). The tool should disappear into the task.

## Anti-references

- Rainbow status/badge palettes; saturated candy colors on light surfaces
- SaaS-cream marketing aesthetics, gradient text, glassmorphism
- A second brand accent (no blue CTA, no teal links — red is the only voice)

## Design Principles

1. Readability first — all text meets WCAG AA contrast on the warm light surfaces
2. One accent, three status signals (green/amber/red); never color alone — pair with icon or label
3. Three surface levels only; flat fills + hairline borders, no drop-shadow decoration
4. Density is welcome; monospace for data values, system fonts for UI
5. Motion conveys state, never decoration

## Accessibility & Inclusion

WCAG AA target. Status always paired with icon/label (colorblind-safe). Respect `prefers-reduced-motion`.
